Requirements

To run Phile requires PHP 5.3.0+. If you're running Apache, mod_rewrite must be enabled.

Installation - Method One: the easy way

  • Extract the latest version of Phile
  • Upload all the files to your server (via FTP or some other deployment)

You can override the default Phile settings (and add your own custom settings) by editing config.php in the root Phile directory. The config.php file lists all of the settings and their defaults. To override a setting simply uncomment it in config.php and set your custom value.

Installation - Method Two: the nerd way

Install composer

curl -s https://getcomposer.org/installer | php

Go to your htdocs directory and download the PhileCMS base distribution with all the dependencies:

cd /your/htdocs/
php /path/to/composer.phar create-project phile-cms/phile

Set up a virtual host inside your apache.conf, and then restart apache:

NameVirtualHost *:80 # if needed

<VirtualHost *:80>
    DocumentRoot "/your/htdocs/"
    ServerName phile.demo
</VirtualHost>

Add an entry to /etc/hosts:

127.0.0.1 phile.demo

EncryptionKey

You must set the encryptionKey value in the config to a non-empty string. There is a generator.php file that, when run, will generate a random string for you.

Permissions

Make sure you have full privileges on the cache folders.

Own the temp/cache folder

chmod 777 temp/cache

Own the lib/cache folder

chmod 777 lib/cache

The temp/cache is needed for the philePhpFastCache. The lib/cache is needed for phileTemplateTwig.

Development

You can turn debug to true when building a theme. It exposes the dump() function in Twig.

Production

Keep the philePhpFastCache plugin enabled. This caches page requests. Which will decrease server load immensely.

Make sure you turn the Twig cache on during production. This will greatly increase the speed of your site.
